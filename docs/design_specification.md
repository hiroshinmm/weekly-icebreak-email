# 詳細設計仕様書 - Weekly Icebreak Email

## 1. 全体構成設計
本システムは、Node.js をベースとしたモジュール型設計を採用している。各モジュールは単一責任の原則に従い、データの取得、生成、処理を担当する。

- **`index.js`**: 全体の実行フローを制御するエントリポイント
- **`src/newsFetcher.js`**: RSS フィードの取得と OGP 画像の抽出
- **`src/insightGenerator.js`**: Google Gemini API（モデル名: `gemini-3.1-flash-lite-preview`）へのプロンプト制御
- **`src/imageProcessor.js`**: 外部 URL 画像の保存および、プレースホルダー管理
- **`src/templateGenerator.js`**: HTML およびメールテンプレートの動的生成
- **`src/emailService.js`**: Nodemailer によるメール配信

## 2. データの流れ (Data Flow)
1. **Fetch (RSS)**: `sources.json` を全件読み込み、各 RSS サイトへ並列にリクエストを送信。
2. **Filter**: カテゴリ別キーワードと、直近 10 日間という日付条件により記事を絞り込む。
3. **OGP Extract**: RSS 内に画像がない場合、記事 URL をフェッチして OGP 画像パスを特定。
4. **AI Generation**: 抽出したニュース記事（タイトル・概要）を Gemini に投げ、日本語翻訳、日本語サマリー（200文字）、エンジニア向け一言考察を JSON 形式で受け取る。
5. **Image Processing**: 画像パスの整理とローカル（docs/output/）への保存。画像がない場合はデフォルト画像を割り当てる。
6. **Output**: Web 表示用の `docs/index.html` の生成、およびメール用の HTML ボディ作成と送信。

## 3. デザイン・UI/UX 設計情報

### 3.1 フォントとタイポグラフィ
- **欧文**: `Inter` (Google Fonts) - 可読性の高いモダンなサンセリフ。
- **和文**: `Noto Sans JP` (Google Fonts) - 美しくバランスの取れた日本語フォント。
- **メインカラー**: `#003399` (インディゴブルー) - プロフェッショナルかつスマートな印象。

### 3.2 Web ギャラリーのデザイン要素
- **カードレイアウト**: グリッド形式でニュースを一枚のカード形式で表示。
- **ホバー・アニメーション**: マウスホバー時にカードの浮き上がり（`box-shadow`）と、画像のわずかな拡大（`zoom`）により、インタラクティブ性を高める。
- **インサイト（Insight）表示**: 目を引くよう、薄いグレー背景にアクセントカラーのボーダーを追加し、ニュース本体と視覚的に分離。

### 3.3 メール配信のデザイン
- **レスポンシブ**: モバイル・デスクトップ両対応の 1 カラム・シングルレイアウト。
- **インラインスタイル**: 多くのメールクライアントでの表示崩れを防ぐため、CSS はすべて HTML 内にインラインで記述されるよう、テンプレート生成時に処理する。

## 4. API モデル戦略 (Gemini)
- **優先順位付きリスト**:
  1. `gemini-3.1-flash-lite-preview`
  2. `gemini-2.0-flash-lite-preview-02-05`
  3. `gemini-2.0-flash`
  4. `gemini-1.5-flash`
- **リトライ制御**: API から 429 エラーが返ってきた場合、メッセージから待機時間を抽出し、自動で待機（Sleep）して再試行する。

## 5. インフラストラクチャ
- **GitHub Actions (`.github/workflows/weekly_icebreak.yml`)**:
  - `cron: '0 0 * * 3'` (毎週水曜日 09:00 JST 頃) に自動実行。
  - 成果物を自動的にリポジトリへコミットし、GitHub Pages へのデプロイをトリガーする。

## 6. 特記事項: `docs/output/` フォルダの役割
本システムの `docs/output/` フォルダには、実行時に取得・リサイズされたニュース画像が保存される。GitHub Pages 上のギャラリー（`index.html`）がこれらの画像を直接参照するため、Web サイトを正常に表示させるには、これらの画像アセットをリポジトリの履歴に含めてプッシュする必要がある。
