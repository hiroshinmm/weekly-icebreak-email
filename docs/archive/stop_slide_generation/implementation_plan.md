# 実装計画: スライド生成の廃止と画像直接利用への移行

## 目的
現在の「A4横サイズのスライド画像をHTML/CSSで生成する」プロセスを廃止し、ニュース元の画像を直接（適切なサイズに調整して）活用する方式に変更します。これにより、メールの軽量化と、元のニュース画像の視認性向上を図ります。

## ユーザーレビューが必要な事項
> [!IMPORTANT]
> スライド形式ではなくなるため、インサイトや要約などのテキスト情報はメール本文の HTML デザインで表現することになります。
> 画像はメール添付に適したサイズ（幅 600px）にリサイズします。

## 変更内容

### ロジック・メールテンプレート (`index.js`)

#### [MODIFY] [index.js](file:///c:/Users/hiros/OneDrive/%E3%83%87%E3%82%B9%E3%82%AF%E3%83%88%E3%83%83%E3%83%97/Antigravity/weekly-icebreak-email/index.js)
- `generateAllSlideImages` 関数を `processNewsImages` に変更。
- Puppeteer を使用して、複雑な HTML スライドを作成する代わりに、ニュース元の画像 (`imageUrl`) を読み込み、幅 600px に固定してスクリーンショット（＝リサイズ）を撮るシンプルな処理に変更。
- `sendEmail` 関数の HTML メールテンプレートを更新し、スライド画像ではなく「リサイズしたニュース画像」を埋め込み、その下にタイトル・要約・インサイトを配置するモダンなレイアウトに変更。
- `docs/index.html` 生成ロジックを、新しく保存された画像を使用するように更新。

## 検証計画

### 自動テスト / 動作確認
- `node index.js` を実行。
- `docs/output/` に生成される画像が、ニュースの元の画像であり、かつ幅が 600px 程度になっていることを確認。
- 送信されたメール（または `docs/index.html`）を開き、画像とテキストのバランスが良いことを確認。
