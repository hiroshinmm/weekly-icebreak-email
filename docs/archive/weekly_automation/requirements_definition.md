# システム要件定義書：週次技術アイスブレイク自動化システム

## 1. プロジェクト概要
ソニー株式会社のディスプレイ・カメラ担当ソフトウェアエンジニアに対して、毎週水曜日の朝に「技術的なインスピレーションと現場の会話の種（アイスブレイク）」を提供する完全自動化システムを構築します。
過去10日間の最新時事情報を収集し、全8つの専門カテゴリーごとにAIによる考察（INSIGHT）と関連画像を複合したスライドを生成、Web上への公開およびメール通知までを一気通貫で行います。

## 2. ターゲットユーザー
- **所属**: ソニー株式会社 
- **職種**: ソフトウェアエンジニア
- **担当領域**: Gaming monitor, Professional monitor, LED display, projector, Spatial reality display (SRD), Camera用モニター・コントローラー等の開発担当者

## 3. システム機能要件

### 3.1. データ収集・フィルタリング抽出（Data Engine）
1. **情報ソース**: 主要テクノロジーメディアのRSSフィードやAPI（例: The Verge, TechCrunch, SonyAlphaRumors, TFTCentral 等）を自動巡回する。
2. **期間フィルタリング**: 実行タイミング（毎週水曜日）から遡って「過去10日間」に公開された最新記事のみを対象とする。
3. **カテゴリー分類**: 取得したニュースをキーワードやAI分類を用いて、以下の「8カテゴリー」に振り分け、最もホットな1トピックを抽出する。
   * SRD / XR
   * Gaming Monitor
   * Production Monitor
   * Camera Control
   * Projector
   * LED Wall Display
   * SONY (ソニー自社の動向)
   * TCL (主要競合の動向)

### 3.2. コンテンツ＆メディア生成（AI Engine & Render Engine）
1. **考察（INSIGHT）の自動生成**: 抽出したニュース記事の概要をもとに、Gemini API等のLLMを使用して「ソニーエンジニアの視点に立った独自の技術考察」を各トピックごとに生成する。
2. **AI関連画像の生成**: ニュース内容およびカテゴリーに合致するプロンプトを動的に生成し、画像生成API（Gemini/DALL-E等）を用いて高品質な関連画像を8枚生成する。
3. **スライドレンダリング**: Puppeteer (Headless Chrome) を用いて、HTML/CSSベースで構成されたA4 Landscapeサイズのスライドを8枚レンダリングし、画像（.png形式等）として保存する。
   * **スライドデザイン要件**:
     * ソニーブランドに相応しい「白基調・プレミアム・清潔感・プロフェッショナル」な2カラムレイアウト。
     * 構成要素: カテゴリータグ、記事タイトル、概要Snippets、「INSIGHT」見出しと考察テキスト、コピー可能な情報ソース文字列＋プレーンなURL、AI生成画像。

### 3.3. Web公開・ホスティング（Hosting System）
1. **静的サイト生成**: 実行完了後、生成された8枚のスライド画像やソースリンク情報を表示するシンプルなWebページ（`index.html`）を生成する。
2. **自動パブリッシュ**: Githubリポジトリに画像およびHTMLを自動コミット＆プッシュする。
3. **GitHub Pages公開**: プッシュされた内容をGitHub Pages経由で即座にWeb公開し、チームメンバーがブラウザから閲覧できる状態にする。

### 3.4. 通知システム（Notification System）
1. **Gmail送信**: システムの全処理完了後、設定された管理者のGmailアドレスへ自動通知メールを送信する（Nodemailer等を使用）。
2. **通知内容**:
   * **Web閲覧リンク**: 「今週のアイスブレイクスライド（公開Web URL）」
   * **サムネイル**: 生成されたスライド画像8枚（メールインライン表示 または 添付）
   * **ソース情報**: 採用された8トピックの元記事タイトルと参照URLのリスト一覧

## 4. 非機能要件・インフラ構成

1. **実行環境**: GitHub Actions
2. **実行トリガー**: `schedule` トリガーによる定期実行（CRON式: `0 0 * * 3` など。毎週水曜日の午前9:00 JST）
3. **プログラミング言語・ランタイム**: Node.js (v18+)
4. **シークレット管理**:
   * APIキー（GEMINI_API_KEY等）やGmail認証情報（GMAIL_USER, GMAIL_PASS）は、コードに直書きせず、GitHub Secrets を用いてセキュアに管理・注入する。
5. **フォールバック**:
   * もし過去10日間に特定カテゴリーの該当ニュースが一切無かった場合は、一般的な当該カテゴリーのAI考察を代替生成するか、スキップしてその旨を表示するエラーハンドリングを実装する。

## 5. 運用・開発フェーズ
本要件を以下のフェーズに分けてアジャイルに新規開発する。
* **Phase 1: 「データ収集 ＆ AI・スライド生成」基盤の開発**（ローカル環境での完全動作確認）
* **Phase 2: 「Web公開 ＆ 通知」機能の開発**（HTML生成・Nodemailer等への連結）
* **Phase 3: 「GitHub Actions」への搭載と最終検証**（クラウド上での定時実行とSecrets管理のテスト）
