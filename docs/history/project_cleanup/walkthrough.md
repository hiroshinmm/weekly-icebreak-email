プロジェクトの不要なファイル・ディレクトリを削除し、ローカル環境をクリーンに保つための設定を行いました。

## 実施した内容

### 1. 不要なファイル・ディレクトリの削除
以下のディレクトリとファイルを削除しました。
- `test_output/`
- `output/` (ルート直下)
- `test_image.js`

### 2. 生成済画像のローカルクリーンアップ
- `docs/output/` 内に存在していた PNG ファイルを全て削除しました。

### 3. `.gitignore` の更新
- `docs/output/*.png` を管理対象外に追加しました。
- これにより、今後ローカルでテスト実行した際に画像が生成されても、Git のコミット履歴を汚すことがなくなります。

## 動作への影響
- **GitHub Actions**: ワークフロー内で `git add -f` を使用しているため、今回の変更（`.gitignore` への追加）によって GitHub 上での画像更新が止まることはありません。
- **Web ギャラリー**: GitHub 上には画像が保持されるため、公開サイトの表示には影響ありません。
