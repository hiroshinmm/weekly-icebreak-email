# 実装計画: AIモデルの修正と取得速度の最適化

## 1. 概要
AI考察生成をより最新かつ適切なモデルで行うように修正し、ニュース取得プロセスを並列化することで、全体のパフォーマンスと信頼性を向上させます。

## 2. 変更内容

### AIモデルの更新 (`src/insightGenerator.js`)
- `MODEL_LIST` の先頭を `gemini-3.1-flash-lite-preview` に変更します。
- 必要に応じて、既存の無効なモデル名を削除またはコメントアウトします。

### ニュース取得の並列化 (`src/newsFetcher.js`)
- `fetchTopics` 内の各ソースに対するループにおいて、`source.urls` を `Promise.all` を使用して並列に取得するように変更します。これにより、直列での取得待ちを解消します。

### 処理フローの効率化 (`index.js`)
- AI考察生成部分の固定的な待機時間（`setTimeout`）を再検討し、モデルの成功率に応じた効率的な処理が可能かを確認します。

## 3. テスト計画
- [ ] 修正後のモデル名で AI が正しいレスポンスを返すことを確認する。
- [ ] 複数のニュースソースが並行して取得されていることをログで確認する。
- [ ] `index.html` がこれまで通り正しく生成されていることを確認する。
