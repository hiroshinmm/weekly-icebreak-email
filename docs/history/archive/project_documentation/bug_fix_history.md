# 不具合修正履歴 - Weekly Icebreak Email

## 主要な不具合と解決策

### 1. AI モデルの 429 エラー (レート制限) およびクォータ切れ
- **不具合内容**: 無料ティアの Gemini API を使用しているため、リクエストが集中した際や1日の制限を超えた際にエラーが発生し、処理が中断されていた。
- **解決策**: 
  - `MODEL_LIST` を導入し、`gemini-3.1-flash-lite`, `gemini-3.0-flash`, `gemini-2.5-flash` などの複数モデルを自動でフォールバックする仕組みを実装。
  - 分間制限に対しては、`retry-after` ヘッダーを解析して一定時間待機後に再試行するロジックを追加。
  - 1日の制限（PerDay）に達した場合は、即座に次のモデルへ切り替えるように改善。

### 2. スライド画像内の画像表示不備
- **不具合内容**: RSS フィードから取得した OGP 画像やサムネイルの URL が無効だったり、取得に失敗した場合、スライドが空白またはエラー表示になっていた。
- **解決策**: 
  - `fetchOgImage` 関数を強化し、複数のメタタグ（og:image, twitter:image）を探索するように改善。
  - Puppeteer のレンダリング時に `onerror` イベントを活用し、画像が読み込めない場合は「No Image Available」のプレースホルダー画像を自動表示するフォールバック機能を実装。

### 3. GitHub Pages での画像表示エラー
- **不具合内容**: GitHub Actions で生成された画像が正確なディレクトリに保存されず、Web ページから参照できていなかった。
- **解決策**: 出力ディレクトリを `docs/output` に固定し、GitHub Actions のワークフローで成果物を正しく push/deploy するように構成を修正。

### 4. コンテンツ末尾の不自然な欠落
- **不具合内容**: AI が生成する要約や考察が、指定された文字数制限により途中で切れてしまうことがあった。
- **解決策**: Gemini のシステムプロンプトにおいて「文章が途中で切れないよう、意味が通る完全な日本語の文章で作成すること」を明示的に指示し、安定性を向上させた。

### 5. カテゴリごとのニュース欠損
- **不具合内容**: 特定の週に特定のカテゴリのニュースが見つからない場合、スライドの枚数が減ったり、エラーが発生していた。
- **解決策**: カテゴリごとに必ず1件の記事を表示することを保証し、ニュースが見つからない場合は「今週の最新ニュースはありませんでした」というプレースホルダーを自動生成するように修正。
